         *displays an error as the Java compiler does not attempt to evaluate the value in a variable when it's used in calcuation, so it doesn't know if the value fits and displays an error.
         If the calcuation uses literal values then Java can figure out the end result at compile time and know whether it fits or not
         such as - byte myNewByteValue = (-128 / 2);
         an int result is being returned from the calcuation in both examples but Java knows the returned value can fit into a byte with the second calcuation.

         *cast the value into a byte by using the following example
         byte myNewByteValue = (byte) (myMinByteValue / 2);
         myNewByteValue ==> -64
         short myNewShortValue = (short) (myMinShortValue / 2);
         myNewShortValue ==> -16384

         ***Java defaults to int
